# Руководство по импорту данных

Поддерживаются форматы **JSON** и **XML**. Кнопки «Импорт из файла» находятся в соответствующих разделах панели администратора.

---

## 1. Пользователи

**Кнопка:** Пользователи → «Импорт из файла»

Создаёт учётные записи пользователей. Если email уже существует — запись пропускается.  
Если `password` не указан — пароль генерируется автоматически (показывается в таблице после импорта, можно скачать Excel).

### Поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `name` | строка | **да** | ФИО |
| `email` | строка | **да** | Email (уникальный) |
| `password` | строка | нет | Пароль (если не указан — генерируется) |
| `role` | строка | нет | Роль: `student`, `teacher`, `staff`, `admin` (по умолчанию `student`) |
| `isPaid` | булево | нет | `true` — платное обучение, `false` — бюджет (по умолчанию `false`) |

> **Важно для XML:** в XML формате пароль обязателен — записи без `<password>` игнорируются.

### JSON
```json
[
  {
    "name": "Иванов Иван Иванович",
    "email": "ivanov@kstu.ru",
    "password": "Пароль123!",
    "role": "student",
    "isPaid": false
  },
  {
    "name": "Петрова Анна Сергеевна",
    "email": "petrova@kstu.ru",
    "role": "student"
  }
]
```

### XML
```xml
<users>
  <user>
    <name>Иванов Иван Иванович</name>
    <email>ivanov@kstu.ru</email>
    <password>Пароль123!</password>
    <role>student</role>
  </user>
</users>
```

**Тестовые файлы:** `users/users.json`, `users/users.xml`

## 2. Группа с участниками

**Кнопка:** Группы → «Импорт из файла»

Создаёт группу и добавляет участников за один запрос.  
Логика для каждого участника:
- Если пользователь с таким email **уже есть** в системе — просто добавляется в группу
- Если пользователя **нет**, но указаны `name` и `password` — создаётся новый пользователь, затем добавляется
- Если пользователя нет и данных недостаточно — запись пропускается с сообщением об ошибке

### Поля группы

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `name` | строка | **да** | Название группы (уникальное) |
| `year` | число | **да** | Год набора |
| `members` | массив | нет | Список участников |

### Поля участника

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `email` | строка | **да** | Email участника |
| `name` | строка | нет | ФИО (нужно при создании нового пользователя) |
| `password` | строка | нет | Пароль (нужен при создании нового пользователя) |
| `role` | строка | нет | Роль (по умолчанию `student`) |
| `isPaid` | булево | нет | Платное обучение (по умолчанию `false`) |

### JSON
```json
{
  "name": "ИВТ-21-1",
  "year": 2021,
  "members": [
    {
      "email": "ivanov@kstu.ru",
      "name": "Иванов Иван Иванович",
      "password": "Пароль123!",
      "role": "student",
      "isPaid": false
    },
    {
      "email": "existing@kstu.ru"
    }
  ]
}
```

### XML
```xml
<group>
  <name>ИВТ-21-1</name>
  <year>2021</year>
  <members>
    <member>
      <email>ivanov@kstu.ru</email>
      <name>Иванов Иван Иванович</name>
      <password>Пароль123!</password>
      <role>student</role>
      <isPaid>false</isPaid>
    </member>
    <member>
      <email>existing@kstu.ru</email>
    </member>
  </members>
</group>
```

**Тестовые файлы:** `groups/group.json`, `groups/group.xml`

## 3. Стипендии

**Кнопка:** Стипендии → «Импорт из файла»

Назначает стипендии студентам по email.  
Если `amount` не указан — берётся из справочника базовых размеров (должен быть задан заранее).  
Студенты на платном обучении (`isPaid: true`) — пропускаются.  
Студенты с задолженностями могут получить только `social`.

### Типы стипендий (`type`)

| Значение | Название |
|----------|----------|
| `academic` | Академическая |
| `social` | Социальная |
| `enhanced_academic` | Повышенная академическая |
| `academic_coeff_1_4` | Академическая (коэф. 1.4) |
| `academic_coeff_1_5` | Академическая (коэф. 1.5) |
| `enhanced_social` | Повышенная социальная |

### Направления (`direction`) — только для `enhanced_academic`

| Значение | Название |
|----------|----------|
| `academic` | Учебная деятельность |
| `research` | Научная деятельность |
| `social` | Общественная деятельность |
| `sports` | Спортивная деятельность |
| `cultural` | Культурная деятельность |

### Поля записи

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `email` | строка | **да** | Email студента |
| `type` | строка | **да** | Тип стипендии (из таблицы выше) |
| `periodStart` | строка | **да** | Дата начала периода (ГГГГ-ММ-ДД) |
| `periodEnd` | строка | нет | Дата окончания (ГГГГ-ММ-ДД) |
| `amount` | число | нет | Сумма в рублях (если не указана — из справочника) |
| `direction` | строка | нет | Направление (только для `enhanced_academic`) |

### JSON
```json
[
  {
    "email": "ivanov@kstu.ru",
    "type": "academic",
    "periodStart": "2025-09-01",
    "periodEnd": "2026-01-31"
  },
  {
    "email": "petrova@kstu.ru",
    "type": "enhanced_academic",
    "direction": "research",
    "periodStart": "2025-09-01",
    "amount": 8500
  }
]
```

### XML
```xml
<scholarships>
  <scholarship>
    <email>ivanov@kstu.ru</email>
    <type>academic</type>
    <periodStart>2025-09-01</periodStart>
    <periodEnd>2026-01-31</periodEnd>
  </scholarship>
  <scholarship>
    <email>petrova@kstu.ru</email>
    <type>enhanced_academic</type>
    <direction>research</direction>
    <periodStart>2025-09-01</periodStart>
    <amount>8500</amount>
  </scholarship>
</scholarships>
```

**Тестовые файлы:** `scholarships/scholarships.json`, `scholarships/scholarships.xml`

## 4. Дисциплины на семестр

**Кнопка:** Группы → выбрать группу → «Дисциплины на семестр» → «Импорт из файла»

Назначает дисциплины группе на конкретный семестр/учебный год.  
Если дисциплина с таким названием не найдена в системе — создаётся автоматически с указанным типом (по умолчанию «Экзамен»).  
Дубликаты пропускаются. В одном файле можно передать данные сразу для нескольких групп.

### Поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `groupName` | строка | **да** | Название группы (должна существовать в системе) |
| `semester` | число | **да** | Номер семестра (1–12) |
| `academicYear` | строка | **да** | Учебный год в формате `ГГГГ-ГГГГ` |
| `disciplines` | массив объектов | **да** | Список дисциплин |

### Поля дисциплины

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `name` | строка | **да** | Название дисциплины |
| `type` | строка | нет | Вид контроля: `exam` — экзамен, `pass_fail` — зачёт (по умолчанию `exam`) |

### JSON
```json
[
  {
    "groupName": "ИВТ-21-1",
    "semester": 3,
    "academicYear": "2025-2026",
    "disciplines": [
      { "name": "Математика", "type": "exam" },
      { "name": "Физика", "type": "exam" },
      { "name": "Программирование", "type": "exam" },
      { "name": "Иностранный язык", "type": "pass_fail" }
    ]
  }
]
```

### XML

Тип дисциплины указывается атрибутом `type` тега `<discipline>`:

```xml
<groups>
  <group>
    <name>ИВТ-21-1</name>
    <semester>3</semester>
    <academicYear>2025-2026</academicYear>
    <disciplines>
      <discipline type="exam">Математика</discipline>
      <discipline type="exam">Физика</discipline>
      <discipline type="exam">Программирование</discipline>
      <discipline type="pass_fail">Иностранный язык</discipline>
    </disciplines>
  </group>
</groups>
```

**Тестовые файлы:** `disciplines/disciplines.json`, `disciplines/disciplines.xml`

## 5. Оценки студентов

**Кнопка:** Оценки → «Импорт из файла»

Выставляет оценки студентам по указанной дисциплине и семестру.  
Если дисциплина не найдена в системе — создаётся автоматически с указанным типом.  
Если оценка для студента уже есть — перезаписывается. В одном файле можно передать оценки по нескольким дисциплинам и группам.

### Поля записи

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `groupName` | строка | **да** | Название группы |
| `disciplineName` | строка | **да** | Название дисциплины |
| `disciplineType` | строка | нет | `exam` или `pass_fail` (по умолчанию `exam`, используется при создании новой дисциплины) |
| `semester` | число | **да** | Номер семестра |
| `academicYear` | строка | **да** | Учебный год в формате `ГГГГ-ГГГГ` |
| `grades` | массив | **да** | Список студентов с оценками |

### Поля оценки студента

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `email` | строка | **да** | Email студента |
| `grade` | строка | **да** | Значение оценки (см. таблицу ниже) |

### Допустимые значения оценок

| Значение | Тип дисциплины | Описание |
|----------|----------------|----------|
| `excellent` | `exam` | Отлично |
| `good` | `exam` | Хорошо |
| `satisfactory` | `exam` | Удовлетворительно |
| `unsatisfactory` | `exam` | Неудовлетворительно |
| `absent_exam` | `exam` | Неявка |
| `passed` | `pass_fail` | Зачёт |
| `failed` | `pass_fail` | Незачёт |
| `absent` | `pass_fail` | Неявка |

### JSON
```json
[
  {
    "groupName": "И-125",
    "disciplineName": "Математика",
    "disciplineType": "exam",
    "semester": 1,
    "academicYear": "2025-2026",
    "grades": [
      { "email": "ivanov@kstu.ru", "grade": "excellent" },
      { "email": "petrova@kstu.ru", "grade": "good" }
    ]
  },
  {
    "groupName": "И-125",
    "disciplineName": "Иностранный язык",
    "disciplineType": "pass_fail",
    "semester": 1,
    "academicYear": "2025-2026",
    "grades": [
      { "email": "ivanov@kstu.ru", "grade": "passed" },
      { "email": "petrova@kstu.ru", "grade": "failed" }
    ]
  }
]
```

### XML
```xml
<gradesList>
  <entry>
    <groupName>И-125</groupName>
    <disciplineName>Математика</disciplineName>
    <disciplineType>exam</disciplineType>
    <semester>1</semester>
    <academicYear>2025-2026</academicYear>
    <grades>
      <grade>
        <email>ivanov@kstu.ru</email>
        <value>excellent</value>
      </grade>
      <grade>
        <email>petrova@kstu.ru</email>
        <value>good</value>
      </grade>
    </grades>
  </entry>
</gradesList>
```

**Тестовые файлы:** `grades/grades.json`, `grades/grades.xml`

## Порядок импорта при первоначальном наполнении системы

1. **Пользователи** — создать всех студентов, преподавателей и сотрудников
2. **Группы** — создать группы и добавить участников (или использовать импорт группы)
3. **Базовые размеры стипендий** — задать вручную через кнопку «Базовые размеры стипендий»
4. **Стипендии** — назначить студентам
5. **Дисциплины** — назначить группам на семестр
6. **Оценки** — выставить оценки студентам по дисциплинам
